# Mid-Level Interview Track

Use this track for AI engineer or mid-level ML engineer interviews.

## What interviewers look for

- Confident model and system trade-off decisions
- Reliable experiment tracking and evaluation strategy
- Production readiness: monitoring, rollback, and incident handling
- Communication clarity across product and engineering stakeholders

## Study checklist

1. Design an experiment plan with success/failure criteria
2. Compare retrieval options (vector DB, keyword, hybrid)
3. Define online and offline metrics for an AI feature
4. Explain how you would debug poor model behavior in production

## Mini practice

- Build: retrieval-augmented QA prototype
- Add: evaluation set with 30-50 curated questions
- Deliverable: architecture note with risk register
