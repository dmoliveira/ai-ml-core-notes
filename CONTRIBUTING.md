# Contributing

Thanks for contributing to AI/ML Core Notes.

## Quick contribution flow

1. Pick one focused improvement (single section or single workflow).
2. Make the smallest useful change.
3. Run validation (`mkdocs build --strict`).
4. Open a PR with clear learner impact and evidence.

## Principles

- Keep content practical and beginner-friendly.
- Explain concepts in simple language first, then add depth.
- Include references to reliable sources.
- Prefer examples that can be adapted to real projects.

## Content format

- Place topic docs under `docs/` in the correct section.
- Add a short summary first, then deeper explanation.
- Add at least one practical example.
- Include links to packages/tools in Python, Julia, and R when relevant.

## Pull requests

- Keep PRs focused and small.
- Add or update links in section indexes.
- Ensure checks pass.

## Release cadence expectations

- Prefer iterative contributions that can ship in frequent small releases.
- Avoid mixing unrelated scope in one PR.
- If you add new guidance, include where it fits in the learning path.

## Validation and evidence

- Include command output snippets or a short pass/fail summary in the PR.
- If a check is skipped, explain why and what was verified instead.
- For docs changes, confirm navigation and cross-links are still coherent.
